Висновки по виконаним завданням

1. Завдання з підрахунком вартості покупки дроїдів (функція `makeTransaction`)

  • У цьому завданні ми реалізували функцію, яка приймає два параметри: кількість замовлених дроїдів та ціну одного дроїда. Функція розраховує загальну вартість покупки та повертає рядок з повідомленням про кількість дроїдів і загальну суму.
  • Основні моменти:
    • Використання шаблонних рядків (`template literals`) дозволяє зручно формувати рядок для виведення результату.
    • Це завдання продемонструвало важливість простих математичних операцій для обчислення загальної вартості.
    • Функція працює ефективно і дозволяє обчислювати загальну вартість покупки на основі заданих вхідних параметрів.

2. Завдання з перевіркою на наявність заборонених слів (функція `checkForSpam`)

  • Функція повинна перевіряти текст на наявність заборонених слів ("spam" або "sale"), і якщо хоча б одне з них знайдено, повертає `true`, в іншому випадку — `false`.
  • Основні моменти:
    • Використання методу `toLowerCase()` дозволяє працювати з текстом незалежно від регістру, що робить функцію універсальною для різних варіантів написання слів.
    • Це завдання акцентує увагу на важливості обробки тексту та реалізації простих перевірок на наявність підстрічок у рядках.
    • Функція ефективно вирішує задачу перевірки і може бути корисною для фільтрації небажаних слів у повідомленнях.

3. Завдання з фільтрацією масиву (функція `filterArray`)

  • Завдання передбачало створення функції, яка приймає масив чисел і значення, а потім повертає новий масив, що містить лише числа більші за задане значення.
  • Основні моменти:
    • Операція фільтрації за допомогою циклу і умовного оператора дозволяє здійснити відбір елементів з масиву.
    • Це завдання є типовим прикладом роботи з масивами та умовами, що часто зустрічаються при розробці функцій для обробки даних.
    • Код є компактним і ефективним, дозволяючи зручно фільтрувати елементи масиву за заданою умовою.

4. Завдання з сортуванням масиву за допомогою швидкого сортування (QuickSort)

  • Тут ми реалізували алгоритм Швидкого сортування (QuickSort), який є ефективним методом сортування для великих обсягів даних.
  • Основні моменти:
    • Швидке сортування працює за принципом "розділяй та володарюй", що дозволяє поділяти масив на менші частини для швидшого сортування.
    • Алгоритм демонструє значення рекурсії у програмуванні та ефективно обробляє масиви з великою кількістю елементів.
    • Це завдання навчило застосовувати алгоритмічні підходи для сортування, що є важливим аспектом у розв'язанні проблем ефективної обробки даних.

5. Завдання з підказками при введенні тексту (автодоповнення)

  • Це завдання передбачало реалізацію підказок, що з'являються при введенні тексту у поле.
  • Основні моменти:
    • Завдання вимагає використання HTML, CSS та JavaScript для реалізації динамічних елементів на сторінці.
    • За допомогою фільтрації масиву підказок на основі введеного тексту можна забезпечити користувачу інтерактивний досвід.
    • Реалізація включала використання подій JavaScript для динамічного оновлення списку підказок при введенні тексту в реальному часі.
    • Це завдання дозволяє зрозуміти, як працюють механізми подій та маніпуляцій з DOM (Document Object Model) у веб-розробці.

Загальні висновки:

  • Математичні операції і логіка: більшість завдань включали використання математичних операцій для обчислень або перевірок, що є важливим аспектом в програмуванні.
  • Маніпуляції з масивами: для ефективної роботи з даними важливо володіти техніками фільтрації, сортування та перебору елементів масивів.
  • Взаємодія з користувачем: підказки при введенні тексту та перевірка вмісту масивів показали, наскільки важливим є правильне проектування інтерфейсів, що взаємодіють із користувачем.
  • Алгоритмічні підходи: виконані завдання вчили працювати з класичними алгоритмами сортування, що є основою для багатьох складних задач з обробки даних.
  • Комбінування HTML, CSS та JavaScript: завдання з автодоповненням показали, як HTML, CSS та JavaScript можуть бути використані разом для створення інтерактивних елементів на веб-сторінках.
